# AI-Powered Skill Gap Analysis

A web application that analyzes a candidate CV against real market demand for a target role. It uses a **4-agent LangGraph pipeline** powered by **Google Gemini** to parse CV content, classify explicit and implicit skills, gather grounded market intelligence, and generate a personalized recommendation report with downloadable PDF output.

## Apps Features

The application consists of seven main areas in a single-page interface.

<p align="center"><img src="images/main_visualization.png" alt="Main Visualization"></p>

---

### 1. CV Upload & Target Role Input

The left sidebar is the control panel for starting analysis.

- **Upload your CV** supports PDF file upload by click or drag-and-drop.
- **Target Role** lets you define the position you want to benchmark against (for example: Data Scientist, AI Engineer, Product Analyst).
- The uploaded file name appears instantly to confirm selected input.
- A **Preview Uploaded CV** button becomes available after successful analysis so users can re-open the original CV file in a new tab.

---

### 2. Analyze Button & Multi-Agent Pipeline Progress

After file and role are filled, click **Analyze** to start the backend pipeline.

<p align="center"><img src="images/visualization_1.png" alt="Sidebar controls and analysis loading"></p>

- The app starts an async analysis job and polls progress from the backend.
- A dedicated loading screen shows 4 pipeline stages:
  1. **CV Parsing**
  2. **Skill Classification**
  3. **Market Analysis**
  4. **Recommendations**
- Each stage displays a heading, label, and subdetail so users know exactly what is running.

---

### 3. Demo Data Mode

For fast preview, users can click **Load Demo Data**.

<p align="center"><img src="images/visualization_keyword.png" alt="Demo mode"></p>

- Demo mode instantly renders Agent 1-3 sections using built-in sample data.
- Agent 4 is then generated by calling the backend recommendation endpoint with demo skills and market intelligence.
- This is useful for testing UI flow when no CV file is available.

---

### 4. Agent 1 - CV Parsing View

The first result section shows structured CV extraction.

<p align="center"><img src="images/visualization_2.png" alt="Agent 1 CV parsing panel"></p>

| Block | Description |
|---|---|
| **CV Summary** | 3-sentence professional summary generated from parsed CV structure |
| **Work Experience** | Role-by-role cards with title, company, period, duration, bullets, and extracted skill tags |
| **Education** | Degree, institution, period, and GPA (if available) |
| **Certifications** | Credential name, issuer, score/year when present |
| **Featured Projects** | Highlighted project list with period and impact description |
| **Organization Experience** | Organization roles and contributions parsed from CV |

---

### 5. Agent 2 - Skills Specialization

The second section classifies skills into explicit and implicit categories.

<p align="center"><img src="images/visualization_3.png" alt="Agent 2 skills section"></p>

| Panel | Description |
|---|---|
| **Skill Summary** | LLM-generated profile covering seniority estimate, sector fit, and achievable roles |
| **Explicit Skills** | Skills explicitly stated in the CV text |
| **Implicit Skills** | Inferred skills from context and experience evidence (hover to see evidence popup) |

---

### 6. Agent 3 - Market Intelligence

The third section displays grounded market research for the selected target role.

<p align="center"><img src="images/visualization_4.png" alt="Agent 3 market intelligence"></p>

| Panel | Description |
|---|---|
| **Job Requirements (Latest Info)** | Source cards containing role overview, key responsibilities, demanded skills, soft skills, tools/software, and certifications |
| **Demanded Skills** | Consolidated high-demand professional skill tags |
| **Soft Skills** | Behavioral and interpersonal skill tags from market signals |
| **Technologies** | Tools, frameworks, platforms, and languages currently requested by employers |
| **Certifications** | Market-relevant certifications aggregated from grounded sources |

---

### 7. Agent 4 - Recommendation Report & PDF Export

The final section converts profile + market gap into actionable recommendations.

<p align="center"><img src="images/visualization_5.png" alt="Agent 4 recommendation report"></p>

- **Summary Analysis** describing overall fit for target role.
- **Skill Gap Analysis** split into:
  - **Market Aligned** skills (already matched)
  - **High Demand / Missing** skills (priority gaps)
- **Key Strengths** cards highlighting strongest profile signals.
- **Resume Action Items (ATS)** showing "Instead of" and "Change to" rewrite suggestions.
- **Estimated Time to Upskill (TTE)** grouped by low, medium, and high effort timelines.
- **Download Report** button exports a polished PDF report generated by backend.


## How to Use the Dashboard

### Step 1 - Open the application

Navigate to `http://localhost:8080` (Docker local frontend) in your browser.

### Step 2 - Upload CV and enter target role

1. Click the upload box (or drag-and-drop) and select a CV in `.pdf` format.
2. Fill **Target Role** with the role you want to evaluate against market demand.

### Step 3 - Start analysis

Click **Analyze** in the sidebar.

The system will run four stages in sequence:

1. Parse CV content
2. Classify explicit and implicit skills
3. Research role demand from grounded web sources
4. Generate recommendation report

### Step 4 - Track pipeline progress

During processing, the loading page shows:

- current stage highlight
- stage heading and activity label
- detailed sub-message describing ongoing work

This gives real-time visibility while backend agents run.

### Step 5 - Review Agent 1 and Agent 2 outputs

After completion, inspect:

- **Agent 1** for structured CV extraction (summary, experience, education, projects, certifications, organizations)
- **Agent 2** for skill profile quality (explicit vs implicit skill mapping and summary)

### Step 6 - Review Agent 3 and Agent 4 outputs

Scroll through:

- **Agent 3** market intelligence source cards and aggregated tags
- **Agent 4** recommendation report (aligned skills, missing skills, strengths, ATS rewrite suggestions, upskilling timeline)

Use the top navigation tabs to jump quickly between sections.

### Step 7 - Download PDF report

Click **Download Report** in the top navigation bar.

The exported `.pdf` file contains a complete formatted report of all four agent outputs:

| Section | Contents |
|---|---|
| **Agent 1 - CV Parsing** | Structured candidate profile and parsed CV blocks |
| **Agent 2 - Skills Specialization** | Explicit/implicit skills and profile summary |
| **Agent 3 - Market Intelligence** | Market demand evidence and skill/technology trends |
| **Agent 4 - Recommendation Report** | Gap analysis, ATS actions, and upskilling plan |

### Credentials setup

This project requires Gemini API credentials.

1. Open `backend/environment.env`.
```bash
   nano backend/environment.env  # or use any text editor
```

2. Fill your actual values:
```env
GEMINI_API_KEY=your_google_ai_studio_api_key_here
MODEL_NAME=gemini-2.5-flash-lite
```

3. Required variables:

| Variable | Description | Required |
|---|---|---|
| `GEMINI_API_KEY` | API key from [Google AI Studio](https://aistudio.google.com/) | Yes |
| `MODEL_NAME` | Gemini model used by backend pipeline | Yes |

> WARNING: Never commit real secrets to source control.

**Example `environment.env` template:**
```env
# backend/environment.env
GEMINI_API_KEY=your_google_ai_studio_api_key_here
MODEL_NAME=gemini-2.5-flash-lite
```


## Code explanation

### Key Backend files

| Category | File | Description |
|---|---|---|
| API Server | `backend/app.py` | FastAPI endpoints for analysis start/status polling, full analyze flow, recommendation-only mode, and PDF report export |
| Pipeline Runner | `backend/main.py` | Builds and runs LangGraph sequence: `read_file -> extract_info -> classify_skills -> search -> report` |
| CV Reader | `backend/config/utils.py` | Reads PDF text with `pdfplumber`, falls back to Gemini vision OCR when needed |
| Agent 1 | `backend/agent_function/agent_1_cv_parsing.py` | Extracts structured CV blocks and generates concise summary |
| Agent 2 | `backend/agent_function/agent_2_specialize_skills.py` | Produces explicit/implicit skill mapping and candidate skill summary |
| Agent 3 | `backend/agent_function/agent_3_market_intelligence.py` | Performs grounded web search, structures per-source market requirements, and aggregates demanded skills/technologies/certifications |
| Agent 4 | `backend/agent_function/agent_4_recommendation_report.py` | Computes aligned/missing skill gap and generates recommendation, ATS rewrites, and upskilling plan |

### Key Frontend files

| Category | File | Description |
|---|---|---|
| Main UI | `frontend/static/index.html` | Single-page layout with sidebar input, loading states, and four result sections |
| Client Logic | `frontend/static/script.js` | Handles upload, async polling, rendering all agent results, demo mode, and PDF download |
| Styling | `frontend/static/style.css` | Full UI design system including responsive layout, cards, tags, progress states, and section styling |
| Reverse Proxy | `frontend/nginx.conf` | Serves frontend static assets and proxies `/api/` requests to backend service |


## Run the project with Docker (Local)

**Prerequisites:** Docker Desktop and a valid Gemini API key in `backend/environment.env`.

**1. Navigate to the project root directory:**

```bash
cd path/to/prod_v1
```

**2. Set backend environment values in `backend/environment.env`:**

```
GEMINI_API_KEY=your_google_ai_studio_api_key_here
MODEL_NAME=gemini-2.5-flash-lite
```

**3. Build and start containers:**

```bash
make up
```

**4. Open in browser:** `http://localhost:8080`

### Makefile reference

| Command | Description |
|---|---|
| `make build` | Build (or rebuild) all Docker images |
| `make up` | Build and start all containers in detached mode |
| `make start` | Start already-built containers |
| `make stop` | Stop running containers |
| `make down` | Stop and remove containers + network |
| `make clean` | Full cleanup (containers, images, volumes, orphans) |
| `make logs` | Tail live logs from all containers |


## Deploy to Google Cloud Run

**Prerequisites:**
- [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- [Google Cloud CLI](https://cloud.google.com/sdk/docs/install) authenticated
- Google Cloud project with Artifact Registry and Cloud Run enabled

---

### One-Time Setup

**1. Set project and region variables:**

```bash
gcloud config set project YOUR_GCP_PROJECT_ID
gcloud config set run/region YOUR_REGION
```

**2. Authenticate Docker with Artifact Registry:**

```bash
gcloud auth configure-docker YOUR_REGION-docker.pkg.dev
```

---

### Deploy / Redeploy Steps

**Step 1 - Prepare production environment values**

Set the backend environment variables in Cloud Run (at minimum):

```
GEMINI_API_KEY = your_google_ai_studio_api_key_here
MODEL_NAME = gemini-2.5-flash-lite
```

**Step 2 - Build images**

```bash
docker build -f backend/Dockerfile  -t YOUR_REGION-docker.pkg.dev/YOUR_PROJECT/YOUR_REPO/skillgap-backend:latest .
docker build -f frontend/Dockerfile -t YOUR_REGION-docker.pkg.dev/YOUR_PROJECT/YOUR_REPO/skillgap-frontend:latest .
```

**Step 3 - Push to Artifact Registry**

```bash
docker push YOUR_REGION-docker.pkg.dev/YOUR_PROJECT/YOUR_REPO/skillgap-backend:latest
docker push YOUR_REGION-docker.pkg.dev/YOUR_PROJECT/YOUR_REPO/skillgap-frontend:latest
```

**Step 4 - Deploy services**

Deploy backend first, then frontend with correct API routing/proxy configuration for your environment.

> Note: This repository currently does not include a ready-to-use Cloud Run multi-container manifest file. Create deployment manifests or CLI commands that match your infrastructure design before production rollout.

**Step 5 - Get the live URL**

```bash
gcloud run services list --platform managed
```
